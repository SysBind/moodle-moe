Future

As Moodle gains in maturity, I hope its directions are strongly influenced by the community of developers and users.

All unfinished features, proposed improvements and feature requests will be gathered on this page.

General

  • Format of existing code should be checked and cleaned up (tabs etc). Define a style guide.
  • Improvements throughout for dealing with very large classes (summary views, pagination, search etc).
  • All table inserts should be converted to use the insert_record function.
  • Full internationalisation of texts using modular "language packs". eg STPhp
  • Implement pop-up help windows all over the site (from tiny help icons)
  • Users can specify their timezone so all times displayed are in local time format.
  • Each course has a private "teacher forum" where teachers keep notes on the course for action research purposes. Messages can be posted here from any module in the course, and they contain links back to their relevant modules.
  • Assessment system: each module can be marked per student. All assessments can be combined on one page.
  • More course formats.
  • Better WYSIWYG editing of texts (using Java or Flash).
  • Make this page more interactive. :-)

Assignment module

  • Module to allow submission and assessment of assignments (to be written).

Chat module

  • Re-implement chat room using Flash, and use user pictures.

Choice module

  • Add capability for more than two choices.
  • Add teacher option of publishing results (ie a poll).

Discussion module

  • Discussion forums need to have abbreviated topic display and pagination for when they get very big.
  • Allow files to be attached to discussion posts (images etc).

Journal module

  • Journal summary index should show teacher remarks.

Survey module

  • Allow teachers to create and share surveys.

Send your additions, suggestions to Martin Dougiamas.
